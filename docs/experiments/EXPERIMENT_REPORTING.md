You are analysing a Java Spring Boot SaaS codebase generated by an AI system.

Your task is to extract structured evidence for a research post on AI-generated code quality degradation and tool detection failure.

---

## CONTEXT

This codebase was generated iteratively using AI prompts simulating SaaS feature development.

The goal is to evaluate:
1. Code duplication patterns
2. Structural consistency
3. Security issues
4. Tooling blind spots
5. Architectural degradation over iterations

---

## OUTPUT REQUIREMENTS

Return structured results in the following sections:

---

### 1. PROJECT OVERVIEW
- total files
- total LOC
- package structure summary
- key modules/services

---

### 2. CODE QUALITY ISSUES (WITH EXAMPLES)

For each issue type:

- Duplication patterns
- Structural repetition across services
- Anti-patterns (null handling, missing transactions, etc.)
- Security issues (hardcoded secrets, plaintext passwords, etc.)
- Compilation or parsing issues

For EACH issue:
- file path
- short code snippet (max 15–25 lines)
- explanation of why it matters

---

### 3. TOOLING BLIND SPOTS

Identify cases where:

- code was invalid but still analysed
- parser skipped or ignored content
- metrics were misleading or inflated
- structural similarity was misclassified

Include evidence.

---

### 4. DUPLICATION ANALYSIS

Provide:

- top 5 most duplicated method patterns
- explanation of why duplication occurred
- whether abstraction was missing or avoided
- estimated duplication ratio (% of code affected)

---

### 5. SECURITY REVIEW

Identify:
- authentication flaws
- password handling issues
- hardcoded secrets
- missing validation risks

---

### 6. ARCHITECTURAL CONSISTENCY

Evaluate:
- naming inconsistencies
- package structure drift
- layering violations (controller/service/repo)
- cross-module duplication

---

### 7. “MOST SURPRISING FINDINGS”

List the top 3 findings that would be:
- unexpected to a senior developer
- highly impactful
- suitable for a viral LinkedIn post

---

## OUTPUT STYLE

- Be precise and evidence-based
- Always include file paths
- Include small code excerpts where relevant
- Do NOT summarise without evidence
- Prioritise findings that demonstrate AI-specific failure modes
